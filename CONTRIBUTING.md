# Contributing to Tibia Toolkit

Thank you for helping improve Tibia Toolkit.

1. Read the [security policy](SECURITY.md) before reporting a vulnerability.
2. Do not include external content, credentials, production configurations, or assets whose redistribution rights are unclear. Tibia-related material must have a completed [CipSoft asset compliance record](docs/CIPSOFT_ASSET_COMPLIANCE.md) before it is added to a public pack or release.
3. Create a branch, make a focused change, and run `npm test`.
4. Open a pull request against `main`. External contributions must be reviewed before merge.
5. Do not modify release workflows, signing policies, installer configuration, or asset-downloader security controls without explaining the impact.
6. Do not contribute cheats, hacks, bypasses, process memory access, injection, input automation, credential capture, or code intended to violate Tibia or platform rules. See the [Tibia fansite agreement](https://www.tibia.com/community/?subtopic=fansites&page=agreement) and [CREDITS.md](CREDITS.md).

By contributing, you agree that your contribution is licensed under GPL-3.0-only.
